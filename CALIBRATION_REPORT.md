## 1. 確認済みの戦闘メカニクス

### 1.1 兵種レーン制の兵士プール方式

戦闘は「兵種レーン制」で行われる。各兵種（盾・槍・弓）がそれぞれ独立したレーンで戦闘し、
集結に参加した全プレイヤーの同兵種兵士がプールされて一つの部隊として扱われる。

### 1.2 英雄の役割

- **英雄はグローバルバフ提供者**であり、英雄vs英雄のマッチアップではない
- **集結主の3英雄**: 全9スキル（各英雄S1/S2/S3）が全レーンにバフを提供
- **乗り手**: S1のみ発動、上位4枠、重複なし、ステータス無関係

### 1.3 ターゲティング順序

前列（盾）→ 中列（槍）→ 後列（弓）の順にターゲットされる。
前列が壊滅するまで中列・後列は直接攻撃を受けない（Ambusher除く）。

### 1.4 三すくみ（殿堂スキルFC3から確認）

| 攻撃兵種 | 有利対象 | ボーナス | 確認元 |
|---------|---------|---------|--------|
| 盾 (FC3) | 弓 | +10% 与ダメ増加 | 殿堂スキルFC3 |
| 槍 (FC3) | 弓 | +10% 与ダメ増加 | 殿堂スキルFC3 |
| 弓 (FC3) | 盾 | +10% 与ダメ増加 | 殿堂スキルFC3 |

**注意:** 槍に対して強い兵種は殿堂スキルからは未確認。現在のモデルでは盾→弓、槍→弓、弓→盾の3関係のみ確定。

### 1.5 槍のAmbusher能力

殿堂スキルFC5により、槍兵は20%の確率で前列（盾）を飛び越えて後列を攻撃できる。

### 1.6 ダメージ式

```
kills = C × sqrt(自軍兵数) × (ATK × Leth) / (敵DEF × 敵HP) × SkillMod × TypeBonus
```

現在のエンジン (`battle-engine.ts`) では暗黙的に **C = 1.0** が使用されている。

### 1.7 損害の按分記録

戦闘での損害は参加者ごとに按分されて記録される。
集結に多くの兵を出したプレイヤーほど、比例して多くの損害を受ける。

### キャリブレーション結果

従来のモデルと実データの比較:

| 状況 | 従来モデル (死亡率) | 実データ推定 | 調整後 |
|------|-------------------|-------------|--------|
| 攻撃側敗北 | 35% | 23-27% | **27%** |
| 防衛側敗北 | 0% | 0% | **0%** |
| 勝利側 | 0% | 3-5% | **4%** |

**結論:**
- 攻撃敗北時の死亡率は35%モデルより低い（推定27%）
- 勝利側は0%ではなく約4%の死亡が発生する
- 防衛側敗北時は従来通り0%死亡（壁による保護）

## 5. 係数Cの議論

### C ≈ 6.87 の推定

| パラメータ | 仮定値 | 根拠 |
|-----------|--------|------|
| 基礎ATK | 1,719 | G10 Lv80 ギアLv30 |
| 基礎DEF | 1,779 | 同上 |
| 基礎Leth | 366 | リファイン5/5/5/5 |
| 基礎HP | 366 | 同上 |
| SkillMod | 1.0 | バフなしと仮定 |
| typeBonus平均 | 1.033 | 1/3有利、2/3等倍 |
| ターン数 | 40 | 典型値 |

### 条件を変えた場合のC推定値

| ターン数 | SkillMod | C推定 |
|---------|----------|-------|
| 20 | 1.0 | ~13.7 |
| 40 | 1.0 | ~6.9 |
| 60 | 1.0 | ~4.6 |
| 40 | 2.0 | ~3.4 |
| 40 | 0.5 | ~13.7 |

C の実質的な推定範囲: **1.0 - 15.0**（広すぎて実用的ではない）

**当面はC=1のまま維持** -- 現在の推定精度では変更するリスクが高い。
SkillModの計算を精緻化することが、Cの推定よりも優先度が高い。

## 7. 残りの課題

### 最優先

| 課題 | 理由 |
|------|------|
| 係数Cの精密な決定にはターン数データが必要 | C推定の最大の不確実要因。ターン数が10-80で変動すると、Cは4-8倍の影響を受ける |
| 特殊強化の個別内訳の完全な解読 | バトルレポートの差分%値から個別の絶対値を復元できないため、SkillModの正確な計算が不可能 |

### 高優先

| 課題 | 理由 |
|------|------|
| FC8/FC10の具体的な数値 | 殿堂スキルの上位効果が未確認。FC3(+10%)やFC5(20%Ambush)は確認済みだが、FC8/FC10は不明 |
| 槍vs盾の相性関係の確認 | 現在、盾→弓、槍→弓、弓→盾は確認済みだが、槍に対して強い兵種が不明。ゲーム内の殿堂スキルまたは公式情報での確認が必要 |

### 中優先

| 課題 | 理由 |
|------|------|
| 両側の追加ステータス絶対値（差分ではなく個別） | SkillMod推定に必須 |
| 防衛側の死亡/負傷/損害の内訳 | 防衛側の死亡率モデル検証 |
| バトル1の防衛側兵力 | データポイント追加 |
| 個別マッチアップのダメージ数値 (IMG_4711-4727の詳細) | 兵種別Cの推定 |
| 同じ英雄構成での複数のバトルレポート | jitter/SkillModの分散推定 |

## 8. 技術メモ

- キャリブレーションスクリプト: `packages/core/src/calibration.ts`
- 戦闘エンジン: `packages/core/src/battle-engine.ts`
- 関数 `calibrate()` でC推定、`analyzeCasualties()` で死亡率検証
- `runCalibration()` でテキスト形式の結果出力
- 死亡率定数は `battle-engine.ts` の `CASUALTY_ATK_LOSER`, `CASUALTY_DEF_LOSER`, `CASUALTY_WINNER` で定義
